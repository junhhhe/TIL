## 정규화(Normalization)와 비정규화(Denormalization)

---

## 1. 정규화 (Normalization)

정규화는 데이터의 **일관성과 무결성**을 확보하고 **데이터 중복을 최소화**하기 위해 테이블을 분리하는 과정

### 목적 및 원칙

| 구분 | 내용 |
| :--- | :--- |
| **핵심 목적** | 데이터 중복(Redundancy) 최소화 및 데이터 무결성(Integrity) 확보 |
| **주요 효과** | 데이터 변경 시 발생하는 **이상 현상(Anomaly) 제거** |
| **주요 원칙** | 정보의 무손실 표현, 분리의 원칙 (하나의 관계성은 하나의 릴레이션으로) |

### 이상 현상 (Anomaly)

정규화를 통해 방지할 수 있는, 데이터 중복으로 인해 발생하는 문제

* **삽입 이상 (Insertion Anomaly):** 원치 않는 정보가 삽입되거나, 자료 부족으로 삽입이 막히는 문제.
* **삭제 이상 (Deletion Anomaly):** 한 정보를 삭제하려다, 연관된 다른 정보까지 손실되는 문제.
* **갱신 이상 (Modification Anomaly):** 중복된 데이터 중 일부만 수정되어 데이터 간 일관성이 깨지는 문제.

### 장점과 단점

| 장점 | 단점 |
| :--- | :--- |
| 데이터 구조의 **안정성 및 무결성 유지** | **JOIN 연산 증가** |
| 이상 현상 제거 | 질의(Query) 응답 시간 저하 (성능 저하) |
| 저장 공간의 최소화 | |

---

## 2. 비정규화 (Denormalization / 반정규화)

비정규화는 **성능 향상**과 **조회 시간 최적화**를 위해, 의도적으로 정규화 원칙을 위배하고 **데이터 중복을 허용**하는 최적화 기법

### 목적 및 필요성

* **핵심 목적:** 읽기(Read) 성능 최적화 및 JOIN 연산 비용 절감.
* **적용 대상:**
    * **지나치게 조인(JOIN)이 많은** 테이블.
    * 자주 사용되는 테이블에 액세스하는 프로세스가 많을 때.
    * 대량의 데이터를 자주 조회하는 경우.

### 장점과  단점

| 장점 | 단점 |
| :--- | :--- |
| **빠른 데이터 조회** (JOIN 비용 감소) | **데이터 갱신/삽입 비용 증가** |
| 조회 쿼리 작성이 간단함 | 데이터 간 **일관성(무결성)이 깨질 위험** 증가 |
| | 더 많은 저장 공간 필요 |

### ⚠️ 주의점

과도한 비정규화는 오히려 데이터 무결성을 해치고 입력/수정/삭제 성능을 늦출 수 있으므로, **성능 이슈가 명확하게 발생했을 때** 제한적으로 적용해야 함

---

## 3. 정규화 vs. 비정규화 비교

| 구분 | 정규화 (Normalization) | 비정규화 (Denormalization) |
| :--- | :--- | :--- |
| **목표** | 데이터 무결성 및 쓰기 성능 최적화 | **조회 성능 최적화** |
| **데이터 중복** | 최소화 (제거) | **의도적으로 허용** |
| **주 사용처** | 일관성이 중요한 OLTP (트랜잭션 처리) 시스템 | 조회/분석이 중요한 OLAP 또는 대규모 시스템 |
